4. Methods

The Exception Reporter has 2 methods. These are described below.

Method Return Type Parameter(s) Description
LoadPropertiesFromConfig Void NONE Causes the Exception Reporter to set it's properties based on values stored in the application's config file. Any properties not specified in the Application's config file are left unchanged. This method provides a way for the Exception Reporter's behaviour to be changed based on the contents of a text file, so that contact information can be changed without the need to recompile the application.

Note that this is a good method of storing and using the Serial Number provided when you purchased the product in order to set the SerialNumber property of the component

Values in the Application's config file must be one of:
  • SLS_ER_SHOW_GENERAL  ("Y" or "N")
  • SLS_ER_SHOW_EXCEPTIONS  ("Y" or "N")
  • SLS_ER_SHOW_ASSEMBLIES  ("Y" or "N")
  • SLS_ER_SHOW_SETTINGS  ("Y" or "N")
  • SLS_ER_SHOW_ENVIRONMENT  ("Y" or "N")
  • SLS_ER_SHOW_CONTACT  ("Y" or "N")
  • SLS_ER_PRINT_BUTTON  ("Y" or "N")
  • SLS_ER_SAVE_BUTTON  ("Y" or "N")
  • SLS_ER_COPY_BUTTON  ("Y" or "N")
  • SLS_ER_EMAIL_BUTTON  ("Y" or "N")
  • SLS_ER_CONTACT_EMAIL
  • SLS_ER_CONTACT_WEB
  • SLS_ER_CONTACT_PHONE
  • SLS_ER_CONTACT_FAX
  • SLS_ER_MAIL_TYPE  ("SMTP" or "SIMPLEMAPI")
  • SLS_ER_SEND_ADDRESS
  • SLS_ER_SMTP_SERVER
  • SLS_ER_SMTP_FROM
  • SLS_ER_SMTP_USERNAME
  • SLS_ER_SMTP_PASSWORD
  • SLS_ER_GENERAL_MESSAGE
  • SLS_ER_EXPLANATION_MESSAGE
  • SLS_ER_CONTACT_MESSAGE_1  (i.e. ContactMessageTop)
  • SLS_ER_CONTACT_MESSAGE_2  (i.e. ContactMessageBottom)
See the properties description page for details of these properties. Note:Property names and "Y" or "N" values are case sensitive.
DisplayException Void Exception Causes the Exception Reporter to be visible. Details of the passed in Exception are displayed.
