5. Integration

Including the Exception Reporter within your application is relatively straight forward, and can be done in one of 2 ways...

Method 1: Placing the component on a form

  1. Download and Install the Exception Reporter
  2. Add a Reference to SLSExceptionReporter.dll to you project
  3. Drag an instance of the Exception Reporter component to your main form (or forms)
  4. Configure the instances properties as appropriate for your application
  5. Add error handling code to use the Exception Reporter on application error (see Example 1)
  6. Include SLSExceptionReporter.dll within your applications distribution package
Note that as an alternative to step 4, you may choose to set property values for the Exception Reporter within your applications config file and make use of the LoadPropertiesFromConfig method...see an example config file here.

NOTE: If you have a valid Serial Number, you must set the Exception Reporters SerialNumber property to be equal to this number in order to stop the Registration Reminders from appearing.

Method 2: Creating and instance at Run-Time

  1. Download and Install the Exception Reporter
  2. Add a Reference to SLSExceptionReporter.dll to you project
  3. Add error handling code to create an Exception Reporter instance at time of Application Error and pass the Exception to the Exception Reporter for display (see Example 2)
  4. Include SLSExceptionReporter.dll within your applications distribution package

NOTE: If you have a valid Serial Number, you must set the Exception Reporters SerialNumber property to be equal to this number in order to stop the Registration Reminders from appearing.

Following are 3 examples, example 1 shows use of an Exception Reporter added to a form, example 2 shows use of an Exception Reporter added at run-time, Example 3 shows use of a wrapper function to wrap the use of the Exception Reporter. The method shown by Example 3 has the advantage that if your use of the Exception Reporter changes, only a single set of code need be updated.

Example 1: Using Component placed on Form

Within your application's code (assuming the instance on the form is called myExceptionReporterInstance.... 
 try {
   //... some code on the form
 }
 catch (Exception ex) {
	// if neccessary call the following method to set the Exception Reporters properties
	// myExceptionReporterInstance.LoadPropertiesFromConfig()
	// alternatively the properties would have already been assigned at design time

	myExceptionReporterInstance.DisplayException(ex);
 }

Example 2: Creating a Component at Run-Time

Within your application's code.... 
using SLSExceptionReporter;
...
...
 try {
   //... some code
 }
 catch (Exception ex) {
	ExceptionReporter myER = new ExceptionReporter();

	// if neccessary call the following method to set the Exception Reporters properties
	myER.LoadPropertiesFromConfig()

	// alternatively you could set the properties programmatically such as...
	myER.ContactWeb = "http://www.zzzmywebsite.com"

	myER.DisplayException(ex);
 }

Example 3: Using a function to wrap the use of the Exception Reporter

Within your application's code.... 


using SLSExceptionReporter;
...
// perhaps create a simple class for this purpose
public class MyStaticClass{
...
	// wrapper function could be a static method of a public class in your app
	public static void MyHandlingFunction(Exception ex) {
		ExceptionReporter myER = new ExceptionReporter();

		myER.LoadPropertiesFromConfig()
		myER.DisplayException(ex);
	}
}
...
...
...
...
//Within other parts of your code simply add
try {
	//your code
} catch (Exception ex) {
	//now use the handling function
	MyStaticClass.MyHandlingFunction(ex);
}
